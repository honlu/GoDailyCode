# 31. 下一个排列

来源：

- LeetCode: https://leetcode.cn/problems/next-permutation/
- CodeTop: 字节跳动高频

## 题意

给定一个整数数组，将它原地改成字典序中的下一个排列。如果当前排列已经是最大排列，则改成最小排列。

## 思路

从右向左找第一个下降点 `i`，满足 `nums[i] < nums[i+1]`。这个位置就是需要变大的最低位。

再从右向左找第一个大于 `nums[i]` 的元素 `j`，交换 `nums[i]` 和 `nums[j]`。交换后，`i` 右侧仍然是降序，需要反转成升序，才能得到“刚好更大”的下一个排列。

容易错的点：

- 全降序时没有下降点，直接反转整个数组。
- 右侧要找的是“从右往左第一个大于 nums[i] 的数”，这样能保证增长最小。
- 题目要求原地修改，不返回新数组。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 复盘

- 题型：数组、双指针、字典序。
- 识别信号：题目要求“下一个排列”、“下一个更大的字典序排列”。
- 面试表达重点：从低位开始寻找可增加的位置，再把后缀调整到最小。
