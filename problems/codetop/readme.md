# CodeTop 面试题

这里记录来自 CodeTop 的公司高频题，目标是为面试准备沉淀可复刷、可测试、可复盘的 Go 题解。

## 目录规范

- 每家公司一个目录，例如 `bytedance/`。
- 每题一个独立 package，目录名优先使用 LeetCode 题号，例如 `31/`。
- 每题至少包含：
  - `readme.md`：题意、思路、复杂度、复盘。
  - `solution.go` 或题名对应的 Go 文件：稳定题解。
  - `solution_test.go` 或题名对应的测试文件：面试级测试用例。
- 函数名尽量保持 LeetCode 原函数名，方便复制到平台验证。

## 做题流程

1. 从 CodeTop 筛选公司和频率，记录到对应公司的进度表。
2. 读题后先判断题型，再写思路，不急着写代码。
3. 先补测试用例，再写实现。
4. 跑通单题测试。
5. 在题目 `readme.md` 里写清楚卡点和下次识别信号。
6. 隔天、三天后、一周后复刷没完全掌握的题。

## 过关标准

- 能在 30 分钟内独立写出。
- 能讲清楚算法正确性。
- 能说出时间复杂度和空间复杂度。
- 能主动指出边界条件。
- 能迁移到同类变形题。
